# 文史哲AI多模块生成系统

## 项目概述
构建一个统一的文史哲AI生成平台，将10个独立功能模块整合为一个可切换的AI应用系统。系统通过统一入口接收用户输入，根据模块类型调用不同的提示词逻辑生成结果。

## 核心规则
- **文档读取及遵循顺序：`prd.md` → `Agent.md`。** 每个任务执行前，必须先理解 PRD 中的产品需求和架构约束，再遵循 Agent.md 中的开发行为规范。
- **每个任务的执行必须遵循 `prd.md` 中的需求和规范。** 包括但不限于：模块接口规范、提示词模板管理、结构化输出要求、通用提示词规范等。任何代码实现都不得偏离 PRD 中定义的架构和约束。

## 技术约束（来自PRD）
- 前端：单页应用（SPA）
- 后端：API服务（Node.js / Python）
- 所有模块必须结构化输出（优先JSON）
- 提示词模板作为独立配置项管理，禁止硬编码在业务逻辑中
- 新增模块无需修改核心代码即可扩展
- 10个模块必须可独立运行
