# 文史哲AI多模块生成系统

## 项目概述
构建一个统一的文史哲AI生成平台，将多个独立功能模块整合为一个可切换的AI应用系统。系统通过统一入口接收用户输入，根据模块类型调用不同的提示词逻辑生成结果。

## 文档体系（按阅读优先级）
1. **`prd.md`** — 产品需求文档，定义多个模块、架构约束、接口规范（开发前必读）
2. **`Agent.md`** — AI开发行为规范，定义32条开发规则（所有任务执行标准）
3. **`CLAUDE.md`** — 本文件，项目技术约束和文档索引
4. **`README.md`** — 项目说明，功能列表和技术栈总览
5. **`API.md`** — 后端API接口文档（多个端点，含请求/响应格式）
6. **`CHANGELOG.md`** — 版本更新日志
7. **`TEST.md`** — 人工测试验证清单
8. **`devlog.md`** — 开发日志（每次任务的修改记录）

## 核心规则
- **文档读取及遵循顺序：`prd.md` → `Agent.md`。** 每个任务执行前，必须先理解 PRD 中的产品需求和架构约束，再遵循 Agent.md 中的开发行为规范。
- **每个任务的执行必须遵循 `prd.md` 中的需求和规范。** 包括但不限于：模块接口规范、提示词模板管理、结构化输出要求、通用提示词规范等。任何代码实现都不得偏离 PRD 中定义的架构和约束。
- **任务完成必须按 Agent.md 第23条输出**：修改文件、修改内容、风险、未完成项、下一步建议。
- **每次任务完成后更新 `devlog.md`**，阶段性保存 Git tag。
- **升级新主要版本后必须同步更新以下文档**：`CHANGELOG.md`（版本日志）、`README.md`（版本号+功能列表+统计数据）、`TEST.md`（测试项+验证结果）、`API.md`（接口变更）、`devlog.md`（开发日志）。

## 技术约束（来自PRD）
- 前端：单页应用（SPA），React + Vite 5 + TypeScript，纯CSS极简风
- 后端：Express + TypeScript API服务
- 所有模块必须结构化输出（优先JSON）
- 提示词模板作为独立配置项管理，禁止硬编码在业务逻辑中
- 新增模块无需修改核心代码即可扩展（自定义模块通过customPrompt机制）
- 10个内置模块必须可独立运行
- 存储：localStorage全量本地存储，无数据库
- 禁止擅自升级/新增依赖，所有依赖变更必须经人工确认
